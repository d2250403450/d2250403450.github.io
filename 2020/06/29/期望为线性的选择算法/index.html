期望为线性的选择算法

这类问题通常被描述为：查找未排序数组中第k大元素或第k小元素。看似这类选择问题要比查找最小或最大值这样的问题更难，但实际上这两类问题的渐进时间复杂度确是相同的，即O(n)。

思路：

可以使用分治算法，使用快速排序算法为模型，但与快排不同，这种算法不会递归处理两边，而只会处理符合选择条件的其中一边。

算法的进入函数和快排类似，其中多了一个nums.size()-k，表明最终需要选择的第k大的元素在数组正确排序顺序中的位置。

quicksort函数中，首先要随机选择一个数，将其归位，获得其数组下标，若该归位将元素下标和目标index相等，说明已经获得第k大元素的数组下标q。

若归位元素下标小于目标index，则说明，目标在返回位置q的右边，则
return quicksort(a, q + 1, r, index)

若归位元素下标大于目标index，则说明，目标在返回位置q的左边，则
return quicksort(a, l, q - 1, index)

随机选择位置后，调用partition函数，将nums[r]归位。

其中遍历 [ l , r-1 ] 区间，设立一个哨兵left==l，表示小于a[r]的字数组的右边界。若存在a[i]<a[r],则交换a[i]和a[left]，并把哨兵left++。最终left指向的位置为一个大于等于a[r]的元素，将其和a[r]j交换，即可把a[r]归位，即最终位置为left。
